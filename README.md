William S. Bainbridge, God from the Machine (2006) **7장 "신앙 발생 모형"**을 NetLogo로 재구현·확장한 행위자 기반 모형(ABM)입니다. 단순한 강화학습 규칙만으로 행위자 집단에서 초자연적 존재에 대한 기대(신앙) 가 자발적으로 발생하는 과정을 시뮬레이션합니다.

개요

베인브릿지 원전은 사이버그(Cyburg)라는 가상 사회의 시뮬레이션 결과만 보고할 뿐, 실행 코드와 환경을 공개하지 않았습니다. 이 저장소는 범용 공개 플랫폼인 NetLogo로 모형을 재구현하여, 작동 방식을 코드 수준에서 투명하게 드러내고 다른 연구자의 검증·후속 분석이 가능하도록 하는 것을 목표로 합니다(원전의 재현 가능성 문제에 대한 대응; cf. Upal 2008). 원전이 명시한 설계는 충실히 옮기되, 원전이 명시하지 않은 미시적 규칙은 코드에서 명시적으로 확정하고 그 내용을 아래에 정리했습니다.

모델이 무엇을 하는가

사이버그에는 네 직업군의 행위자가 삽니다: 에너지(노랑)·물(파랑)·식량(초록)·산소(청록) 생산자. 각 행위자는 다섯 자원(에너지·물·식량·산소·생명)과, 다섯 개의 3×4 학습 행렬을 가집니다. 학습 행렬의 세 행은 "세상에 집단이 몇 개 있는가"에 대한 세 이론(2집단·3집단·4집단)을, 각 칸은 해당 집단과의 거래 경험의 누적 가치를 나타냅니다(초기값 500–999 난수).

매 틱(tick)마다 각 행위자는:


소비 — 다섯 자원을 1씩 소비(하나라도 0 이하가 되면 사망).
생산 — 직업군에 따라 자원 생산. 에너지 생산자는 에너지를 직접 만들고, 나머지는 상위 자원을 소비해 자기 자원을 생산.
거래 탐색 — 가장 부족한 자원을 정하고 → 학습 행렬에서 분산이 큰 이론을 골라 → 목표 집단을 정한 뒤 → 무작위 상대를 만나 상대 식별자에 모듈로(modulo) 연산을 적용해 소속을 추정. 원하는 집단이면 거래를 시도(성공 시 관련 학습 칸 +250, 실패 시 −100, 최솟값 50).


신앙은 어떻게 발생하는가. 생명(life) 자원은 특별합니다. 어떤 행위자도 영생을 줄 수 없어 자연적 집단과의 거래는 항상 실패(−100)합니다. 그러나 행위자가 이론이 상정한 실제 집단 수를 넘어서는 '잉여(초자연적) 칸' 을 선택하면, 실제 상대가 존재하지 않아 거래도 학습 신호도 발생하지 않습니다. 자연적 칸은 시간이 지나며 깎이지만 잉여 칸은 초기 난수값을 유지하므로, 잉여 칸의 상대적 가치가 점점 커지고 행위자는 점점 더 자주 잉여 집단을 영생의 원천으로 택하게 됩니다 — 이것이 신앙(faith) 의 발생입니다.

신앙도(faith score) 는 한 행위자가 생명을 구할 때 잉여 칸을 선택할 확률로 정의되며, 인구 평균 신앙도는 전 행위자의 산술 평균입니다.

식별자와 모듈로 분류. 각 행위자는 직업에 따라 ID mod 4로 구분되는 식별자를 받습니다(에너지 ≡0, 물 ≡1, 식량 ≡2, 산소 ≡3). 행위자는 상대 ID를 이론의 집단 수(2·3·4)로 나눈 나머지로 상대의 소속을 추론합니다. 이 방식 덕분에 4집단 이론(객관적 진실)에서는 소속이 실제 직업과 정확히 일치합니다.

요구 사항


NetLogo 7.0 이상 (모델이 .nlogox XML 형식으로 저장됨 — 이 형식은 NetLogo 7.0에서 도입).
무료 배포: https://ccl.northwestern.edu/netlogo/


실행 방법


NetLogo에서 모델 파일(.nlogox)을 엽니다.
setup — 초기 인구를 생성합니다.
go — 시뮬레이션을 진행합니다(인터페이스의 그래프·모니터로 신앙도 변화를 실시간 확인).


대량 실험(다중 시드, 인구 규모 스윕 등)은 BehaviorSpace(Tools → BehaviorSpace)를 사용하세요. 본 연구의 결과도 BehaviorSpace로 산출했습니다.

상수는 setup-globals에서 수정합니다.

두 실행 조건


재생산 OFF (use-reproduction? = false) — 사망한 행위자를 대체하지 않아 인구가 점차 소멸합니다(약 1,000틱 내 전멸). 베인브릿지 7장의 원설정에 해당합니다.
재생산 ON (use-reproduction? = true) — 사망한 자리를, 무작위로 선택된 두 부모의 학습 행렬을 상속받은 새 행위자로 즉시 충원합니다(행 단위로 분산이 더 큰 부모의 행을 통째 복사; 8장 가족 상속 방식). 인구가 유지되어 장기 동학을 관찰할 수 있으며, 본 연구가 추가한 확장입니다.


주요 출력 지표 (reporter)


mean-faith — 인구 평균 신앙도
mean-faith-energy-role / -water-role / -food-role / -oxygen-role — 직업별 평균 신앙도
mean-life-theory2-prob / -theory3-prob / -theory4-prob — 이론(2·3·4집단) 선택 확률
mean-natural-life-prob / mean-supernatural-life-prob — 자연적 vs 초자연적 영생 기대
faith-stdev, faith-percentile-10, faith-percentile-90 — 신앙도 분포
wealth-faith-correlation — 부(wealth)와 신앙의 상관
trade-success-rate, births-this-tick, deaths-this-tick, role-deaths-* 등 — 거래·인구 동학


재현 결과 (요약)


신앙의 출현: 인구 평균 신앙도가 약 0.79–0.80 수준으로 발생하여, 베인브릿지가 보고한 0.793과 사실상 일치.
엘리트–대중 비대칭: 노동이 필요 없어 부가 높은 에너지 생산자(엘리트)의 신앙도가 가장 높게 형성.
단순성 편향: 객관적 진실인 4집단 이론이 약화되고, 더 단순한 2집단 이론이 우세해짐.
규모 불변성: 인구를 1,000·2,000·5,000명으로 달리해도 위 결과가 사실상 동일(민감도 분석).


구현상의 결정 (투명성 노트)

원전이 명시하지 않아 본 재현에서 명시적으로 확정한 규칙의 예:


생명(life) 자원의 초기값은 500–1,000 사이 난수.
거래 시 요청자는 가장 풍부한 자원 50단위를 주고 가장 부족한 자원 50단위를 받으며, 양쪽 모두 줄 수 있을 때만 성사.
학습 강화는 성공 +250, 실패 −100(하한 50).
재생산 시 메모리 상속은 부모 두 행 중 분산이 큰 행을 복사(베인브릿지 8장 방식).
trim-excess-population: 인구 상한 초과 시 부(wealth) 최하위부터 제거하는 안전장치(사망/충원이 균형을 이루는 정상 실행에서는 거의 발동하지 않음).



참고: v10에서는 성능을 위해 상류/중류/하류 계급(wealth class) 관련 정렬 연산 코드를 제거했습니다. 직업군은 그대로 유지됩니다.
